# Проектирование и настройка корпоративной сети «Сфера»

📌 **Описание проекта**
Реализация сетевой инфраструктуры для двух отделов (Маркетинг и HR) и серверной зоны. Проект выполнен в эмуляторе с упором на сегментацию трафика и безопасность внешних соединений.

### 📋 Техническое задание (Цели)
* **Масштабируемость:** расчет подсетей с запасом адресов для роста отделов.
* **Безопасность:** изоляция сегментов с помощью VLAN (VLAN 10, 20, 30).
* **Маршрутизация:** настройка выхода в интернет (SNAT) и публикации сервисов (DNAT).
* **Стандартизация:** разработка и внедрение конвенции имен для оборудования.

### 🛠 Стек технологий и инструменты
* **Cisco IOS:** настройка роутинга, коммутации и NAT.
* **VLAN (802.1Q):** логическая сегментация сети.
* **Draw.io:** проектирование логической топологии.
* **Wireshark:** анализ сетевого трафика и отладка DHCP.

## 🏗 Логическая топология и адресация
Сеть построена на базе адресного пространства `192.168.60.0/24`. Для оптимизации и безопасности использованы следующие подсети:

* **VLAN 10 (Маркетинг):** `192.168.60.0/26` (до 62 хостов)
* **VLAN 20 (HR):** `192.168.60.64/26` (до 62 хостов)
* **VLAN 30 (Серверная):** `192.168.60.128/28` (до 14 хостов, статические IP)

## 🛠 Техническая реализация
1. **DHCP и статика:** Маршрутизатор `Router-ceftik-ya-ru` настроен как DHCP-сервер для отделов. Серверы (VLAN 30) используют статические адреса (например, Web-сервер — `192.168.60.130`).
2. **NAT:** * **SNAT:** выход в интернет через публичный IP `203.0.113.5`.
   * **DNAT (Port Forwarding):** публикация веб-сервера (порт 80) во внешнюю сеть.
3. **Коммутация:** Настроены Trunk-каналы и Access-порты для разделения трафика.

### 📋 План коммутации
| Устройство | Порт | Режим | VLAN / Назначение |
| :--- | :--- | :--- | :--- |
| **Router** | e0/2 | Outside | Подключение к провайдеру (203.0.113.1) |
| **Router** | e0/1 | Inside | Trunk-канал до коммутатора |
| **Switch** | e0/0 | Trunk | Канал до маршрутизатора |
| **Switch** | e0/1 – e0/30 | Access | VLAN 10 (Маркетинг) |
| **Switch** | e1/1 – e1/25 | Access | VLAN 20 (HR) |
| **Switch** | e2/1 – e2/3 | Access | VLAN 30 (Серверная) |

### 🏷 Конвенция имен
В проекте реализована строгая система именования устройств согласно ТЗ:
* **Маршрутизатор:** `Router-ceftik-ya-ru`.
* **Коммутатор:** `Switch_Office`.
* **Конечные точки:** `PC_Marketing_01`, `PC_HR_01`, `Srv-Web`.
